### 校园活动管理系统（前后端不分离）

### 一、系统概述

### 1.1 系统定位

校园活动管理系统是一个面向高校活动组织与管理的平台，采用 Spring Boot + Thymeleaf 前后端不分离架构，实现活动发布、活动报名、用户管理等核心业务功能。

系统主要服务于：

* 管理员
* 普通用户

通过 Spring Security 实现身份认证与权限控制。

### 1.2 技术架构

| 层次     | 技术选型              |
| ---------- | ----------------------- |
| 前端     | Thymeleaf + Bootstrap |
| 后端     | Spring Boot           |
| ORM      | MyBatis-Plus          |
| 安全框架 | Spring Security       |
| 缓存     | Redis                 |
| 消息队列 | RabbitMQ              |
| 数据库   | MySQL                 |
| 构建工具 | Maven                 |

### 二、角色与权限设计

系统采用 RBAC 权限模型。

### 管理员（ADMIN）

拥有全部权限：

* 用户管理
* 活动管理
* 报名管理
* 数据统计
* 文件管理

### 普通用户（USER）

拥有个人相关权限：

* 用户注册
* 用户登录
* 浏览活动
* 活动报名
* 取消报名
* 查看我的报名
* 上传个人头像
* 修改个人信息
* 修改密码

### 三、系统功能设计

系统划分为五个核心模块。

### 模块一：用户认证与用户管理

| 功能                 | 权限       |
| ---------------------- | ------------ |
| 用户注册             | 游客       |
| 用户登录             | 所有用户   |
| 用户退出             | 所有用户   |
| 修改密码             | 已登录用户 |
| 修改个人资料         | 已登录用户 |
| 用户列表             | 管理员     |
| 新增用户             | 管理员     |
| 修改用户             | 管理员     |
| 删除用户（逻辑删除） | 管理员     |

### 业务规则

* 用户名唯一
* 密码采用 BCrypt 加密
* 注册默认角色 USER
* 删除采用逻辑删除

### 模块二：活动管理

| 功能                 | 权限     |
| ---------------------- | ---------- |
| 活动列表             | 所有用户 |
| 活动详情             | 所有用户 |
| 发布活动             | 管理员   |
| 修改活动             | 管理员   |
| 删除活动（逻辑删除） | 管理员   |
| 上传活动图片         | 管理员   |

### 查询功能

支持：

* 活动名称搜索
* 活动地点搜索
* 活动时间筛选
* 分页查询

### 模块三：活动报名管理

| 功能         | 权限     |
| -------------- | ---------- |
| 活动报名     | 普通用户 |
| 取消报名     | 普通用户 |
| 我的报名     | 普通用户 |
| 报名记录管理 | 管理员   |

### 业务规则

* 同一活动不可重复报名
* 人数达到上限后停止报名
* 活动开始后不可取消报名
* 删除采用逻辑删除

### 模块四：首页统计分析

| 功能           | 权限   |
| ---------------- | -------- |
| 用户总数统计   | 管理员 |
| 活动总数统计   | 管理员 |

| 数据可视化统计 | 管理员 |

### 数据展示

首页展示：

* 用户总数

使用 ECharts 展示：

* 活动报名情况统计图

### 模块五：文件上传管理（加分拓展项）

| 功能         | 权限     |
| -------------- | ---------- |
| 上传用户头像 | 普通用户 |
| 上传活动图片 | 管理员   |

* 使用 Spring Boot MultipartFile 实现文件上传
* 文件保存到服务器本地目录
* 数据库中保存文件路径
* 支持 jpg、png 等图片格式

### 六、RabbitMQ消息通知（加分拓展项）

### 功能场景

* 用户报名成功后发送通知
* 用户取消报名后发送通知
* 活动即将开始时发送提醒通知

### 实现方式

1. 用户完成报名操作
2. 系统向 RabbitMQ 发送消息
3. 消息消费者异步处理通知
4. 在“我的通知”页面展示消息
